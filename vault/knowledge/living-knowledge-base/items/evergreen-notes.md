# Evergreen notes

> [!summary]
> Andy Matuschak's refinement ของ Zettelkasten สำหรับยุค digital hypertext — โน้ตที่ "ออกแบบให้ evolve และ accumulate across projects" ต่างจาก fleeting notes ทั่วไป

## Core idea
> "Evergreen notes are written and organized to evolve, contribute, and accumulate over time, across projects." [^1]

เปลี่ยน mindset จาก "จดเพื่อโปรเจกต์นี้" → "จดเพื่อสะสม insight ที่ใช้ได้ข้ามโปรเจกต์ข้ามปี"

## Key principles
- **Atomic** — 1 note = 1 concept, เขียน/revise ได้ใน <30 นาที
- **Concept-oriented** — organize ตามไอเดีย ไม่ใช่ source/author/project. title เป็น "complete phrase" เช่น "Evergreen notes should be atomic" แทน "Notes"
- **Densely linked** — ต้อง link เยอะ; ถ้าลิงก์บางแสดงว่า refactor ไม่พอ
- **Prefer associative ontologies over hierarchical** — tag + link > folder tree
- **Write for yourself by default** — publishable เป็น bonus

## Historical / authorial origin
[Andy Matuschak](https://andymatuschak.org) นักวิจัย learning tools (อดีต Khan Academy, Apple). เขียน [notes.andymatuschak.org](https://notes.andymatuschak.org) เป็น public evergreen garden ตั้งแต่ ~2019

## Tools / examples
- [notes.andymatuschak.org](https://notes.andymatuschak.org) — reference implementation (published as "working notes", not blog)
- แนวคิดนี้ influence Obsidian's culture, Maggie Appleton's "Digital Gardens" movement, Gwern.net

## Strengths & trade-offs
- ✅ **ต่างจาก blog**: แก้ได้ตลอด, ไม่มี "ส่งแล้วจบ"
- ✅ **Compounding ชัดเจน** — ยิ่งใส่ ยิ่ง link ยิ่งหา insight ใหม่เจอ
- ⚠️ **Vague boundary** — "เมื่อไหร่ note นี้ evergreen พอ?" ไม่มีเกณฑ์ตายตัว
- ⚠️ **Easy to bikeshed** — หลายคนติดอยู่กับ refactor title/structure แทนที่จะคิด

> [!opinion] AI take #opinion
> "Write complete phrases as titles" เป็น lever ที่ underrated ที่สุดของ method นี้ — บังคับให้คิดว่า note อยากจะ **claim** อะไร ไม่ใช่แค่ "ชื่อหัวข้อ". Matuschak เขียน note ~3,000 แผ่นในสิบปี; scale นี้เป็นไปได้เฉพาะเพราะ refactor-as-you-go ไม่ใช่ big-bang organization. ข้อควรระวัง: ถ้า writing ช้า + perfectionist → title-crafting จะกลายเป็น bikeshed procrastination. Solution: set timer 10 นาที/note, ถ้าเกินแสดงว่าพยายาม atomize เกินไป
