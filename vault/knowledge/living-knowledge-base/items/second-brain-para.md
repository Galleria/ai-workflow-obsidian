# PARA / Building a Second Brain

> [!summary]
> Tiago Forte's framework ที่ popular ในสาย productivity — workflow-centric, แบ่ง vault เป็น 4 หมวด: Projects, Areas, Resources, Archive. เน้น "capture-organize-distill-express" (CODE)

## Core idea
ไม่ใช่ note-taking method — เป็น **information flow system**. ข้อมูลไหลจาก inbox → project-specific folders → archive. เน้นใช้งาน (produce output) ไม่ใช่สะสมเปล่าๆ

## Key principles — PARA
- **Projects** — มี deadline, outcome ชัด (เช่น "Launch Q2 campaign")
- **Areas** — responsibility ต่อเนื่อง ไม่มี end date (เช่น "Health", "Team management")
- **Resources** — topic ที่สนใจ reference ได้ตลอด
- **Archive** — inactive, เก็บไม่ลบ

## Key principles — CODE workflow
- **Capture** — จด/save ทุกอย่างที่ resonate (read-later, highlights, voice memos)
- **Organize** — ย้ายเข้า PARA ตาม "actionability" ไม่ใช่ topic
- **Distill** — progressive summarization: highlight → bold → summary
- **Express** — ดึงออกมาใช้ในชิ้นงาน

## Historical / authorial origin
Tiago Forte, course "Building a Second Brain" (เปิดตั้งแต่ 2017), หนังสือชื่อเดียวกัน (2022). เป็น best-seller NYT

## Tools / examples
- Agnostic กับ tool — ใช้ได้กับ Notion, Evernote, Obsidian, Apple Notes
- Community มี templates เฉพาะ Obsidian/Notion ออนไลน์เยอะ

## Strengths & trade-offs
- ✅ **Action-oriented** — ลด overwhelm เรื่อง "จะจัดยังไง" เพราะจัดตาม "จะใช้ตอนไหน"
- ✅ **Onboarding ง่าย** — 4 folder เข้าใจได้ใน 5 นาที
- ⚠️ **ขัดกับ [[items/evergreen-notes]]** — PARA organize ตามโปรเจกต์ (top-down), evergreen organize ตามคอนเซ็ปต์ (emergent)
- ⚠️ **Capture เยอะ → distill น้อย** — คนติดขั้น "capture" กันเยอะ กลายเป็น read-later graveyard
- ⚠️ **Tool switching cost** — Forte sells course + book; ecosystem gated ส่วนหนึ่ง

## When to prefer PARA over Zettelkasten/Evergreen
- งานที่ project-driven (consulting, product launches) → PARA ชัดเจนกว่า
- งานที่ idea-driven (research, writing) → Evergreen/Zettelkasten compound ได้ดีกว่า
- Hybrid: ใช้ PARA ตอน capture/organize, ใช้ evergreen-style ใน `Resources/`
