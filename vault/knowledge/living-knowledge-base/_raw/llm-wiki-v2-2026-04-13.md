## LLM Wiki v2 ระบบความจำ AI ที่ต่อยอดจากแนวคิดของ Karpathy

เมื่อวานหลังจากแชร์การใช้ Obsidian ไปมีน้องท่านหนึ่ง หลังไมค์ส่งข้อความมา พูดคุยแลกเปลี่ยนเรื่อง LLM Wiki ผมเองได้เรียนรู้อะไรหลายประเด็นใหม่ๆจากบทสนทนานี้เหมือนกัน

สำหรับคนที่ยังไม่เคยได้ยิน ผมแนะนำสั้นๆก่อนเกี่ยวกับ LLM Wiki มันคือแนวคิดและระบบที่ให้ AI (Large Language Models) ทำหน้าที่ สร้างและจัดการฐานความรู้องค์ความรู้ของตัวเอง ในรูปแบบ wiki เพื่อง่ายต้องการเรียนรู้ เข้าใจ และเรียกใช้องค์ความรู้ด้วย AI ในอนาคตต่อไป แทนที่จะต้องอนุมานหรือค้นหาข้อมูลเดิมซ้ำๆ ทุกครั้งที่ถูกถาม เสริม ความจำระยะยาว, ลดการหลอน และเพิ่มประสิทธิภาพการทำงานลดการทำงานซ้ำๆ โดย แนวคิดนี้ริเริ่มโดย Andrej Karpathy (อดีตหัวหน้าฝ่าย AI ของ Tesla และนักวิจัยของ OpenAI) และได้รับความสนใจอย่างมากในชุมชน AI

โพสนี้ที่พูดถึง LLM Wiki เพราะพึงมีการนำไปต่อยอดอัพเดต V2 เข้าในใหม่และแก้ไขข้อจำกัดก่อนหน้าโดยเปลี่ยน wiki เรียบๆ ให้กลายเป็น ระบบความจำที่มีชีวิต สิ่งที่เพิ่มเข้ามาสรุปสั้นๆได้แก่

1. **Confidence Scoring** — ทุกข้อมูลมีคะแนนความน่าเชื่อถือ

2. **Memory Tiers** — แบ่งเป็น 4 ระดับ: Working Memory (สิ่งที่เพิ่งเกิดขึ้น), Episodic Memory (สรุปแต่ละ session), Semantic Memory (ความรู้ข้ามสาย), และ Procedural Memory (ขั้นตอนการทำงาน) — แต่ละระดับยิ่งกระชับและอยู่ได้นานขึ้น

3. **Knowledge Graph** — ไม่ใช่แค่หน้าแบน ๆ ที่เชื่อมด้วยลิงก์ แต่เป็น entity และ relationship ที่มีประเภท เช่น "A เป็นสาเหตุของ B ยืนยันโดย 3 แหล่ง ความน่าเชื่อถือ 0.9" การค้นผ่าน graph จับความเชื่อมโยงที่การค้นหาด้วยคีย์เวิร์ดทั่วไปมองข้ามไป

4. **Hybrid Search** — BM25 + Vector + Graph Traversal

5. **Automated Hooks** — เมื่อมีแหล่งใหม่ → นำเข้าอัตโนมัติ / สิ้นสุด session → บีบอัดและจัดเก็บ / ตามกำหนดเวลา → ตรวจสอบ รวมข้อมูล และลดความสำคัญ งานบ้านที่ฆ่า wiki ทั่วไปถูกจัดการครบ

6. **Forgetting Curves** — ข้อมูลที่ไม่ถูกเข้าถึงหรือเสริมเป็นเวลานานจะค่อย ๆ จางลง ไม่ใช่การลบทิ้ง แต่เป็นการลดลำดับความสำคัญ — การตัดสินใจด้านสถาปัตยกรรมจางช้า, บั๊กชั่วคราวจางเร็ว

7. **Contradiction Resolution** — AI สามารถเข้าใจและแก้ไขข้อขัดแย้งโดยอัตโนมัติ
